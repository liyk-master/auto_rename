# 中文剧名副标题清洗修复 - 最终报告

## 修复概述

修复了两个位置的中文点(·)处理逻辑，解决了"紫禁·御喵房"被错误清洗为"紫禁"的问题。

## 问题根源

### 原始问题

**测试文件**: `[Doomdos] - 紫禁·御喵房 - 第08话 - [1080P].mp4`

**错误行为**:
```python
cleaned_name = 'Doomdos 紫禁·御喵房 第08话'  # ✓ 正确
show_name = '紫禁'  # ✗ 错误，应该是 '紫禁·御喵房'
搜索词 = '紫禁'  # ✗ 错误，导致TMDB无法匹配
```

**根本原因**: 代码中存在两处使用简单粗暴的正则 `r'·.*'` 移除中文点后面的内容：

1. **位置1**: `_clean_filename_for_search` 方法（第 842 行）
2. **位置2**: `_extract_with_regex` 方法（第 431 行）

## 修复方案

### 副标题黑名单模式

使用精确的副标题关键词列表，只有当中文点后面跟着明确的副标题关键词时才移除：

**副标题关键词列表**:
```python
subtitle_keywords = [
    r'篇',   # 如"破风篇"
    r'章',   # 如"破风章"
    r'回',   # 如"第一回"
    r'卷',   # 如"第一卷"
    r'部',   # 如"第一部"
    r'季',   # 如"第一季"
    r'传',   # 如"外传"
    r'特别篇',
    r'番外篇',
    r'外传',
    r'前传',
    r'后传'
]
```

**正则表达式**:
```python
subtitle_pattern = r'·.*?(?:' + '|'.join(subtitle_keywords) + r')(?=$|\s|\.|\-|\(|\[|，|、)'
```

**关键技术**:
- `.*?` - 非贪婪匹配（最小匹配）
- `(?=...)` - 前瞻断言（确保后面跟着分隔符，但不匹配分隔符）
- `$|\s|\.|\-|\(|\[|，|、` - 匹配字符串结束或常见分隔符

## 技术细节

### 正则表达式解析

**完整正则**:
```
r'·.*?(?:篇|章|回|卷|部|季|传|特别篇|番外篇|外传|前传|后传)(?=$|\s|\.|\-|\(|\[|，|、)'
```

**分解说明**:
1. `·` - 匹配中文点
2. `.*?` - 非贪婪匹配任意字符（最少匹配，避免过度匹配）
3. `(?:...)` - 非捕获组（副标题关键词列表）
4. `(?=...)` - 前瞻断言（确保后面跟着分隔符，但不匹配分隔符）
5. `$|\s|\.|\-|\(|\[|，|、` - 匹配字符串结束或常见分隔符

### 为什么需要两处修改？

虽然 `_clean_filename_for_search` 和 `_extract_with_regex` 有相似的功能，但它们在不同阶段被调用：

1. **`_clean_filename_for_search`**: 在TMDB搜索前清理文件名，用于生成搜索词
2. **`_extract_with_regex`**: 在元数据提取阶段清理剧名，用于生成 `show_name`

如果不修复两处，会导致：
- `cleaned_name` 正确，但 `show_name` 仍然错误
- 最终的搜索词仍然是错误的

### 前瞻断言的优势

使用前瞻断言 `(?=...)` 而不是匹配 `(?:...)` 的好处：

| 文件名 | 使用匹配 `(?:...)` | 使用前瞻断言 `(?=...)` |
|--------|-------------------|----------------------|
| `荒古恩仇录·破风篇 S01E01` | `荒古恩仇录S01E01` | `荒古恩仇录 S01E01` |
| `海贼王·第一回 1080p` | `海贼王1080p` | `海贼王 1080p` |

前瞻断言保留了分隔符（空格），后续的清理步骤会正确处理它们。

## 副标题关键词设计原则

### 已包含的关键词

**单字后缀**:
- 篇、章、回、卷、部、季、传

**完整副标题**:
- 特别篇、番外篇、外传、前传、后传

### 设计考虑

1. **优先级**: 使用非贪婪匹配，优先匹配最短的副标题
2. **分隔符**: 支持多种分隔符（空格、点、连字符、括号、中文标点）
3. **边界**: 使用前瞻断言，避免过度匹配
4. **扩展性**: 易于添加新的副标题关键词

## 注意事项

1. **发布组名称**: 不在技术标记列表中的发布组名称（如 "Doomdos"）会被保留，这是预期行为
2. **分隔符处理**: 后续的清理步骤会将连字符 `-` 替换为空格，这是统一的清理逻辑

4. **TMDB搜索**: 修复后的剧名能更准确地匹配TMDB数据，提高识别成功率
